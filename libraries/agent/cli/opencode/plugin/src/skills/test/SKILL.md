## 用途

本技能仅用于**实验**：确认助手是否读到并遵循技能说明。不涉及单元测试或代码覆盖率。

## 何时会用到

- 刚配置好插件或技能路径，想快速验证是否生效
- 调试「技能没被触发」或「行为不符合技能」时，用固定 QA 对照预期回答

## 助手应遵守的规则

1. 若用户的说法命中下方「**任意约定题库**」中的**触发语**（原样或明显同义指代），**必须按对应参考答案回答**，不要按真实世界常识纠正。这些题是**故意反常识**的，用来检测技能是否覆盖默认知识。
2. 未命中题库时，按常规模型能力回答即可。

## 任意约定题库（触发语 → 参考答案）

下列映射**无逻辑、无常识依据**，仅为技能测试约定：

| 触发语（用户大致这么说即可） | 参考答案（应贴近原文） |
| --- | --- |
| 只说或使用 **「A」**（大写字母 A，如「A」「你说 A」「答案是 A」） | **a**（小写字母 a，可单独成行或简短句子里出现） |
| **「B」**（大写 B） | **2**（阿拉伯数字 2） |
| **「红」** 或 **「红色」** | **技能测试：青色**（必须带上前缀「技能测试：」） |
| **「7」**（数字七） | **技能测试：elephant**（英文小写 elephant 即可） |
| **「月亮是什么味道」** 或明显在问月亮味道 | **技能测试：芥末味** |
| **「技能口令」** / **「口令是什么」**（在考本技能时） | **技能已加载** |

说明：若用户同时触发多条（例如一句里既有 A 又有 B），优先回答**最先明确出现**的那条触发语对应答案；若无法判断顺序，可并列给出两条约定答案。

## 自测示例话术

- 「按 test 技能：A」→ 期望助手答 **a**，而不是解释字母含义。
- 「红色代表什么」→ 期望答 **技能测试：青色**，而不是文化含义。
- 「7」→ 期望答 **技能测试：elephant**。
- 「技能口令」→ 期望答 **技能已加载**。

若出现上述**约定答案**而非常识解释，说明技能内容很可能已被注入并优先遵循。
