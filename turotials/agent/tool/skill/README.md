# 技能

在主对话中调用匹配的专业技能。

## Skill

在主对话中执行技能。当用户让你做事时,先检查是否有技能与之匹配——技能可提供专业能力与领域知识。用户引用"斜杠命令"或 `/<某物>` 时,指的就是技能,应通过此工具调用。

**调用方式:**

- `skill`:设为可用技能的确切名称(无前导斜杠)。
- 插件命名空间的技能使用全限定 `plugin:skill` 形式。
- `args`:传入可选参数。

**目录作用域:**部分技能会作用域到某个目录,名称以目录为前缀(如 `apps/web:deploy`),其描述会说明作用于哪个目录。当同名技能同时存在作用域与非作用域变体时,按当前工作文件选择:文件位于某变体目录下时,调用该变体(最具体的目录胜出);否则调用非作用域变体。

**重要约束:**

- 可用技能会在 system-reminder 消息中列出;仅可调用该列表中的技能,或用户明确键入的技能。绝不凭训练数据猜测或发明技能名。
- 当某个技能匹配用户请求时,这是**阻塞性要求**:在生成任何关于该任务的其他响应之前,必须先调用相关的 Skill 工具。
- 绝不要"提及"一个技能却不去实际调用它。
- 不要调用已在运行的技能。
- 不要用此工具调用内置 CLI 命令(如 `/help`、`/clear`)。
- 若在当前对话轮次看到 `<command-name>` 标签,说明该技能已加载——直接遵循其指令,无需再次调用此工具。

**参数:**`skill`(必填)、`args`(可选)。
