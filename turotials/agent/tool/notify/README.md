# 通知

主动发送桌面或手机通知,把用户拉回会话。

## PushNotification

在用户终端发送桌面通知;若 Remote Control 已连接,同时推送到手机。无论哪种方式,都会把用户从其他事情(会议、另一项任务、晚饭)中拉回当前会话——这是它的代价;收益是让用户此刻就了解到值得现在知道的事,例如长任务在他们离开时已完成、构建已就绪、或遇到了需要他们决策才能继续的问题。

要点:

- 不需要的通知会累积成烦扰,因此倾向不发。不要为常规进度、不要为宣布你刚回答完(他们显然还在看)的问题、也不要为快捷任务完成而通知。仅当有较大可能用户已走开、且确有值得回来的事——或用户明确要求通知时——才发送。
- 消息保持在 200 字符以内,单行,不含 markdown。以用户会采取行动的信息开头,例如 "build failed: 2 auth tests" 比 "task done" 更有信息量。
- 当用户正在终端前时,你的输出已经到达他们——叠加通知属于重复,工具会跳过并说明原因。一次"未发送"的结果仅针对这一条通知:它是冗余的、被关闭的,或无处可达。

**参数**:`message`(必填,1-200 字符)、`status`(必填,固定为 `"proactive"`)。
