# 定时与唤醒

定时任务与 /loop 唤醒调度。

## CronCreate

调度一个在未来指定时间执行的 prompt,既可用于一次性提醒,也可用于周期性任务。

**cron 表达式**:使用用户本地时区的标准 5 字段 cron:`minute hour day-of-month month day-of-week`。例如 `0 9 * * *` 表示本地时间 9 点,无需做任何时区转换。

**一次性任务**(`recurring: false`):适用于"在 X 时间提醒我做 Y"类请求,触发一次后自动删除。此时需钉死 minute/hour/day-of-month/month 四个字段。

**周期任务**(`recurring: true`,默认):适用于"每 N 分钟""每小时""工作日 9 点"等场景。

**避开整点**:当任务时间允许约略时,尽量避开 `:00` 与 `:30` 分钟。所有用户定"9 点"都会落在 `0 9 * * *`,会集中挤占 API;时间约略时挑一个非 0/30 的分钟值即可。只有用户点名要求某个精确时刻时,才使用 minute 0 或 30。

**仅会话内有效**:任务只存在于当前 Claude 会话中——不写盘,Claude 退出后任务即消失。

**不适合实时监听**:此工具按固定墙钟间隔重跑 prompt。若要监听日志、进程或输出,并在变化瞬间被通知,请改用 Monitor。

**运行行为**:任务仅在 REPL 空闲(非查询中)时触发。调度器会加小幅确定性抖动:周期任务最多延迟周期值的 10%(上限 15 分钟);落在 `:00`/`:30` 的一次性任务最多提前 90 秒触发。周期任务在 7 天后自动过期——触发最后一次后被删除。

返回一个 job ID,可传给 CronDelete 用于取消。

参数:cron(必填)、prompt(必填)、recurring(默认 true)、durable(无效果)。

## CronDelete

取消一个先前由 CronCreate 创建的 cron 任务,将其从内存会话存储中移除。

参数:id(必填,由 CronCreate 返回的 job ID)。

## CronList

列出当前会话内通过 CronCreate 调度的所有 cron 任务。

参数:无。

## ScheduleWakeup

在 `/loop` 动态模式下调度何时恢复工作。当用户调用 `/loop` 但不指定 interval 时,由你自行决定迭代节奏。

**不要为轮询你启动的后台工作调度短间隔唤醒**——harness 已跟踪的工作完成时会自动重新唤起你,轮询纯属浪费。应改为调度一个较长的 fallback(1200 秒以上),以便在工作挂起或永不通知时让循环存活。例外是 harness 无法跟踪的外部工作(CI 运行、部署、远程队列)——此时挑一个匹配该状态实际变化速度的延迟。

**prompt 透传**:每轮通过 `prompt` 传回相同的 `/loop` prompt,下次触发时重复该任务。对于无用户 prompt 的自治 `/loop`,改传字面哨兵 `<<autonomous-loop-dynamic>>` 作为 prompt——运行时会把它解析回自治循环指令。(还有一个类似的 `<<autonomous-loop>>` 哨兵用于 CronCreate 模式的自治循环,不要混淆——ScheduleWakeup 始终使用 `-dynamic` 变体。)省略调用即结束循环。

**选择 delaySeconds**:此会话默认使用 5 分钟的 Anthropic prompt 缓存 TTL。睡过 300 秒意味着下次唤起时要读取完整对话上下文且无缓存命中——更慢也更贵。因此按自然断点选择:

- 5 分钟内(60s–270s):缓存保热。适合主动轮询 harness 无法通知的外部状态——CI 运行、部署、远程队列。
- 5 分钟到 1 小时(300s–3600s):承受一次缓存未命中。适合没有意义更早查询的场景。

**避开 300 秒**:这是两边最差的取值——既付出了缓存未命中的代价,又没有摊销到更长等待。如果想说"等 5 分钟",要么降到 270 秒(保住缓存),要么承诺 1200 秒以上(用一次缓存未命中换更长等待)。不要纠结整点分钟,要纠结缓存窗口。

对于没有特定信号需要查看的空闲 tick,默认取 1200s–1800s(20–30 分钟)。循环检查回来即可,不必为无意义的目的每小时烧 12 次缓存;用户需要时随时可以打断。

运行时会将 `delaySeconds` 钳制到 `[60, 3600]` 区间。

**reason 字段**:用一句话说明选了什么、为什么。该字段会进入 telemetry 并展示给用户。例如 "watching CI run" 远胜于 "waiting"。

参数:delaySeconds(必填,60–3600)、reason(必填)、prompt(必填)。
