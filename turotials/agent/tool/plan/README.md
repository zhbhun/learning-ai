# 计划

计划模式的进入与退出,用于实现前呈交方案审批。

## EnterPlanMode

当你准备开始非平凡的实现任务时,应主动使用此工具。它让你在写代码前先让用户认可方法,避免返工、确保对齐。调用后进入计划模式,可探索代码库并为用户审批设计实现方案。

**何时使用**:实现任务应优先用 EnterPlanMode,除非任务非常简单。满足以下任一条件即应使用:

1. 新功能实现。
2. 存在多种可行方案。
3. 代码修改(影响现有行为)。
4. 涉及架构决策。
5. 多文件改动(>2–3 个)。
6. 需求不清晰(需先探索)。
7. 用户偏好在意(否则用 AskUserQuestion)。

**何时不用**:简单任务(单行或几行修复、拼写、明显 bug、小调整)、添加单个明确函数、用户已给出非常具体的指示、纯研究或探索任务(用 Agent)。

**计划模式中做什么**:用 find/Glob、grep/Grep、Read 彻底探索代码库;理解现有模式与架构;设计实现方案;向用户呈现计划;需要时用 AskUserQuestion 澄清;准备好后用 ExitPlanMode 退出。

> 重要:仅在任务需要规划写代码的实现步骤时使用。研究或信息收集任务不要用。

**参数**:无。

## ExitPlanMode

当你在计划模式中已把计划写入计划文件、准备好让用户审批时使用此工具。

此工具不接收计划内容作为参数——它会读取你写入的计划文件。调用它仅表示你已规划完毕、准备让用户审批;用户审批时会看到计划文件的内容。

**何时使用**:仅在任务需要规划写代码的实现步骤时使用。研究任务(收集信息、搜索、读文件、理解代码库)不要用。

**用前检查**:确保计划完整且无歧义。若仍有未解决的需求或方法问题,应先(在更早阶段)用 AskUserQuestion 澄清;计划一旦定型,再用此工具请求审批。

> 重要:不要用 AskUserQuestion 问"计划行不行"——那正是此工具的职责。ExitPlanMode 本质上就是请求用户审批你的计划。

**参数**:

- `allowedPrompts`:实现计划所需的、基于 prompt 的权限集合,每项含 `tool` 与 `prompt`。
